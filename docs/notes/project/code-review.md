
# code review

### code review 内容
- 代码规范（eslint 不能全检查，eg：变量命名、代码语义）
- 重复代码要抽离、复用
- 单个函数内容过长，需要拆分
- 算法复杂度是否可用？是否可继续优化
- 是否有安全漏洞
- 拓展性如何？（不用为了拓展而拓展，不封闭即可）
- 是否和现有功能重复？
- 是否有完善的单元测试？

### code review 时机
- 提交PR（或MR）时，通过代码 diff 进行 code review
- 每周例行一次集体 code review

### 持续优化
- 每次 code review 的问题要记录下
- 归纳整理，形成自己的代码规范体系
- 新加入成员要提前学习，提前规避
